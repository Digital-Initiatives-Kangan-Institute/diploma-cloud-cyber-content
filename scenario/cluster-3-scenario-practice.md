# Cluster 3 — Scenario (Practice)

> **Role:** Practice · **Vehicle:** Website · per `scenario-flow.md`.
>
> Lean cluster-and-role delta — see `scenario-flow.md`, `cluster-3-scenario-assessment.md`, and the website ICT records. Meta/design doc.

## 1. Engagement

Students rehearse the **cloud-infrastructure improve-loop** on the **website** — taking its 2023 **single-AZ pilot** baseline and improving it across security, reliability, scalability, and cost — before applying the same loop, team-led and assessed, to Ledgerline. The website's deliberately un-hardened pilot is rich in improvable surface, which makes it an ideal practice substrate.

## 2. Scenario deltas

- **Maximum improvable surface:** the single-instance, single-AZ pilot has obvious gaps in every dimension (no load balancer, no autoscaling, no Multi-AZ, no CDN, no WAF, no DR) — so each of security / reliability / scalability / cost has clear, concrete practice targets.
- **Individual rehearsal** of the improvement techniques; the **team-leadership** layer is assessed on Ledgerline, not practised here.
- **Continuity, no leakage:** students met the website as the CL2 *assessment* vehicle and now *improve* it here — same system, different task. Nothing leaks: the CL2 deliverable is a DR plan, not an improvement, and the website is never assessed in CL3.

## 3. Vehicle state

The website's **2023 single-AZ baseline** — the Website Cloud Architecture — Baseline Design, the Website Infrastructure Specifications, and the website's ICT state. The improved target states are the student's practice output.

## 4. Practised focus

Rehearses **ICTCLD504** improvement techniques on the website, ahead of the team-led assessment on Ledgerline.
